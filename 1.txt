"请帮我彻底优化这个底部的音乐播放控制条 UI，现在的样式太扁平、太生硬，缺乏现代感和质感。目标是打造精致、有呼吸感、符合现代暗黑美学的播放控件（参考 Apple Music、Spotify 或网易云音乐的高级播放页）。
现状分析：
进度条太细，滑块（圆点）太大且颜色突兀，与轨道缺乏融合。
按钮图标太细（线性），在深色背景下不够清晰，缺乏点击反馈。
整体像一个扁平的贴片，没有立体感和深度。
时间显示太紧凑，不够易读。
具体优化方案：
1. 进度条（核心交互区）
轨道粗细：将进度条轨道加粗（例如从 4px 增加到 6px 或 8px），高度适当增加。
轨道样式：轨道默认状态使用深灰/半透明黑（如 rgba(255,255,255,0.2)），已播放部分使用渐变色（如紫蓝色渐变 linear-gradient(to right, #8B5CF6, #6366F1)）或纯净的亮白色，增加高级感。
滑块（Thumb）：
缩小滑块尺寸（不要太大），改为环形/空心圆或带内阴影的实心圆，颜色与已播放轨道一致。
滑块周围加一层极淡的发光效果（如 box-shadow: 0 0 10px rgba(139, 92, 246, 0.5)），让它看起来有“悬浮感”。
增加磁吸感：当拖动滑块时，它应该能吸附到最近的时间点（如果有 LRC 歌词，可以吸附到歌词时间点）。
时间显示：
时间文字（3:01 / 3:22）字号加大（16px），颜色提亮（白色或浅灰），左右间距增加，避免拥挤。
可选：在进度条下方加一层细微的进度条阴影（模拟深度）。
2. 播放控制按钮
图标风格：将细线图标改为填充图标或粗线性图标（更清晰），颜色与进度条已播放部分一致（如紫色/白色）。
中间播放/暂停按钮：
放大：中间的播放/暂停按钮要比左右按钮大（例如直径 56px vs 40px），形成视觉焦点。
背景：给它加一个圆形渐变背景（如从深紫到浅紫），或者毛玻璃效果（backdrop-filter: blur），让它从深色背景中“浮”出来。
悬停/点击反馈：鼠标悬停时背景变亮，点击时有轻微缩放（scale 0.95）和颜色加深。
左右按钮（上一/下一）：
缩小一点，与中间按钮形成对比。
悬停时图标颜色变亮，点击时有轻微位移（translateX 或 translateY 0.5px）。
3. 整体容器与布局
圆角与阴影：播放条容器使用大圆角（如 20px 或 24px），加一层极淡的底部阴影（box-shadow: 0 -4px 20px rgba(0,0,0,0.3)），让它看起来像是“浮”在页面底部，而不是贴死。
背景：如果背景是透明的，建议加一层极淡的深色渐变背景（如 linear-gradient(to top, rgba(0,0,0,0.8), rgba(0,0,0,0.2))），增强可读性。
内边距：增加内部 padding（如 16px），让元素之间呼吸感更强，不要太挤。
4. 动效与微交互
进度条加载：当歌曲加载时，进度条轨道显示一个从左到右的加载动画（如 shimmer 效果）。
按钮动效：所有按钮在 hover 和 click 时都要有流畅的过渡（transition: all 0.2s ease）。
滑块拖动：拖动滑块时，进度条应有平滑的跟随动画（不要卡顿）。
技术实现要求：
使用原生 CSS 或 Tailwind CSS，确保代码结构清晰。
滑块交互使用 input[type="range"]自定义样式，或用 div 模拟（更灵活）。
背景毛玻璃效果使用 backdrop-filter: blur(10px)+ background: rgba(255,255,255,0.05)。
颜色方案：主色建议用紫蓝色系（#8B5CF6, #6366F1）或青色系（#06B6D4），与歌词高亮色保持一致。
